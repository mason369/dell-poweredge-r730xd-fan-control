# 安全说明

语言：中文 | [English](SECURITY.en-US.md)

本项目会通过 iDRAC/IPMI 直接控制 Dell R730xd 风扇。安全边界既包括凭据保护，也包括硬件操作安全。请把它当作可以影响服务器温度和稳定性的本地管理工具，而不是只读监控面板。

## 安全模型

- 应用运行在当前 Windows 用户会话中。
- 应用连接到用户配置的 iDRAC/BMC 地址。
- 应用通过内置 `ipmitool.exe` 执行 IPMI over LAN 命令。
- 应用不会把密码提交到仓库。
- 应用不会把密码放入命令行参数。
- 应用不会在 IPMI 命令失败时静默切换到其他后端或伪造成功状态。
- 应用会把真实失败原因显示在界面状态栏和最近日志中。

## 凭据保存

默认情况下，密码只存在于当前运行进程的 UI 输入框和内存中。用户可以选择“使用 DPAPI 保存密码”，此时应用会使用 Windows DPAPI 的 `DataProtectionScope.CurrentUser` 加密密码，并把密文写入本地设置文件：

```text
%LocalAppData%\DellR730xdFanControlCenter\settings.json
```

DPAPI 密文与当前 Windows 用户上下文绑定。复制到其他用户或其他机器后通常无法解密。删除本地设置文件可以移除保存的主机、用户名、密码密文、预设和界面设置。

## 命令可见性

命令执行层使用：

```text
ipmitool -I lanplus -H <host> -U <user> -E <ipmi-arguments>
```

密码通过 `IPMI_PASSWORD` 环境变量传给 `ipmitool -E`。这样做的目的是避免密码出现在命令行参数、任务管理器命令行列、进程枚举输出或界面日志中。

界面日志会记录：

- 执行的 `ipmitool` 路径和参数。
- 退出码。
- 成功或失败状态。
- 命令耗时。
- 失败时的 stdout/stderr 摘要。

界面日志不会记录密码。

## 本地日志

应用内“最新日志”保留最近 80 条运行日志。启动阶段发生未处理异常时，应用会写入：

```text
%LocalAppData%\DellR730xdFanControlCenter\startup-error.log
```

提交 issue、截图或日志前，请检查其中是否包含你的 iDRAC 地址、用户名、主机名、路径或其他不希望公开的环境信息。不要公开 iDRAC 密码、VPN 地址、内网网段、安全策略、资产编号或其他敏感信息。

## iDRAC 权限

本应用需要能读取 SDR 并发送 Dell OEM raw IPMI 命令的 iDRAC 账号。受限账号可能出现以下情况：

- `mc info` 成功，但 `raw 0x30 ...` 控制命令失败。
- `sdr elist` 成功，但风扇模式切换失败。
- 命令返回权限或认证错误。

建议为此应用使用最小必要权限的 iDRAC 账号，并避免复用高价值管理密码。若你不确定权限边界，请先在隔离网络或测试窗口中验证。

## 网络边界

IPMI over LAN 应只暴露在可信管理网络中。不要把 iDRAC/IPMI 直接暴露到公网。建议：

- 使用独立管理 VLAN 或受控内网。
- 限制能访问 iDRAC 的来源 IP。
- 使用强密码并定期轮换。
- 关闭不需要的管理协议。
- 在不可信网络中不要保存密码。

## 操作安全

风扇控制会改变服务器散热能力。手动低转速可能导致 CPU、硬盘、PCIe 卡、电源、内存或机箱内部温度升高。每次调速后请持续观察：

- CPU 温度和 CPU 使用率。
- Inlet / Exhaust 温度。
- Fan 1-6 RPM。
- 硬盘、背板、线缆在位和冗余状态。
- 功耗、电压、电流。
- iDRAC 告警和系统事件。

当服务器负载未知、环境温度较高、硬盘数量较多、传感器异常或你无法持续观察时，应使用 Dell 自动模式。

## 单风扇 raw target 风险

单风扇控制默认关闭。本机 R730xd/iDRAC 2.82 实测 Fan 1 target byte `0x00` 没有单独控制 Fan 1，而是让全部风扇高转。不同固件可能有不同映射。启用单风扇控制前必须确认：

- 你的 iDRAC 固件接受该 raw 命令。
- target byte 与实际风扇一一对应。
- 操作后 RPM 和温度符合预期。
- 能随时切回 Dell 自动模式。

如果行为不符合预期，请立即恢复 Dell 自动模式。

## 软件恒温策略风险

软件恒温策略依赖 BMC SDR 传感器读取和 CPU 温度识别。当前逻辑优先使用名称包含 `CPU` 的温度传感器；如果没有 CPU 命名行，则使用所有温度传感器中的最高值。找不到温度传感器会报错。

该策略不是固件级实时控制回路。它受以下因素影响：

- iDRAC 返回 SDR 的速度。
- 网络延迟。
- 命令超时设置。
- 轮询间隔设置。
- 传感器命名和固件输出格式。

达到紧急温度阈值时，应用会发送 Dell 自动模式命令，让 BMC 接管风扇。该动作本身仍依赖 IPMI 命令能成功执行。

## 供应链与内置资产

仓库包含内置 `ipmitool.exe`、Cygwin DLL 和 ECharts 前端资产。发布或重新打包时需要确认：

- 内置二进制来自可信来源。
- 文件没有被替换或篡改。
- `Assets/Charts/echarts.LICENSE.txt` 和 `Assets/Charts/echarts.NOTICE.txt` 随资源保留。
- 发布包包含 `BundledTools/ipmitool/**` 和 `Assets/Charts/**`。

如果你替换 `ipmitool.exe` 或图表资产，请在文档中记录来源、版本、校验方式和兼容性变化。

## 失败处理原则

本项目要求失败显式暴露：

- 缺少内置 `ipmitool.exe` 时直接报错。
- iDRAC 主机、用户名、密码为空时直接报错。
- 命令超时时终止进程并显示超时原因。
- `ipmitool` 返回非零退出码时显示 stdout/stderr。
- SDR 输出为空时直接报错。
- 缺少 UI 翻译时直接报错。

除非未来有明确设计并在文档中说明触发条件、用户可见行为和风险，否则不要加入会掩盖真实失败的静默 fallback、默认成功、吞异常或假数据。

## 报告安全问题

报告问题时请包含：

- 应用版本或提交号。
- Windows 版本。
- iDRAC 固件版本。
- 触发步骤。
- 可公开的错误消息。
- 相关日志中已经脱敏的片段。

请不要包含：

- iDRAC 密码。
- 未脱敏的内网地址规划。
- VPN、堡垒机或防火墙策略细节。
- 私有资产编号或主机名。
