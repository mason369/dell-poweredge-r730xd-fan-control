# 安全说明

语言：中文 | [English](SECURITY.en-US.md)

本项目会通过 iDRAC/IPMI 直接控制 Dell R730xd 风扇。安全边界既包括凭据保护，也包括硬件操作安全。请把它当作可以影响服务器温度和稳定性的本地管理工具，而不是只读监控面板。

## 安全模型

- 应用运行在当前 Windows 用户会话中。
- 应用连接到用户配置的 iDRAC/BMC 地址。
- 应用通过内置 `ipmitool.exe` 执行 IPMI over LAN 命令。
- 应用不会把密码提交到仓库。
- 应用不会把密码放入命令行参数。
- 应用不会在 IPMI 命令失败时静默切换到其他后端，也不会把失败状态记录为成功。
- 应用会把真实失败原因显示在界面状态栏和最近日志中。

## 凭据保存

默认情况下，密码只存在于当前运行进程的 UI 输入框和内存中。用户可以选择“使用 DPAPI 保存密码”，此时应用会使用 Windows DPAPI 的 `DataProtectionScope.CurrentUser` 加密密码，并把密文写入本地设置文件：

```text
%LocalAppData%\DellR730xdFanControlCenter\settings.json
```

DPAPI 密文与当前 Windows 用户上下文绑定。复制到其他用户或其他机器后通常无法解密。删除本地设置文件可以移除保存的主机、用户名、密码密文、预设和界面设置。

设置保存采用同目录临时文件和替换流程：完整 JSON 先写入唯一的 `.settings-<GUID>.tmp`，强制刷盘后再替换 `settings.json`。写入、刷盘或替换失败会中止当前保存，不会先截断活动设置文件；应用也不会把保存显示为成功。异常断电或强制终止仍可能留下不包含当前生效状态的临时文件，但不会被启动流程当作设置加载。启动遇到损坏 JSON、空设置或非法预设时，应用保留原文件，停在设置页显示并记录根因，不启动轮询或风扇自动策略。该行为保护文件完整性，但不提供设置版本历史或自动备份；删除或覆盖 `settings.json` 前仍应由用户自行备份。

## 命令可见性

命令执行层使用：

```text
ipmitool -I lanplus -H <host> -U <user> -E <ipmi-arguments>
```

密码通过 `IPMI_PASSWORD` 环境变量传给 `ipmitool -E`。这样做的目的是避免密码出现在命令行参数、任务管理器命令行列、进程枚举输出或界面日志中。

最近日志和 JSONL 运行日志会记录：

- 执行的 `ipmitool` 路径和参数。
- 退出码。
- 成功或失败状态。
- 命令耗时。
- 用户命令、传感器刷新和软件恒温 tick 的起止时间、耗时、`operationId` 和终止状态。
- 失败时界面显示的错误信息；`ipmitool` 失败时错误文本可能包含 stdout/stderr 摘要。

最近日志和 JSONL 运行日志不会记录密码，但可能记录 iDRAC host、用户名、工具路径、本机路径、预设名、传感器名称和命令参数。

## 本地日志

应用内“最新日志”保留最近 80 条内存日志。总览页提供“打开日志文件夹”按钮，打开：

```text
%LocalAppData%\DellR730xdFanControlCenter\logs
```

运行日志文件按本地日期命名：

```text
%LocalAppData%\DellR730xdFanControlCenter\logs\runtime-YYYYMMDD.jsonl
```

每一行都是一个完整 JSON 对象，不把一条事件拆成多行。常见字段包括 `eventId`、`timestamp`、`level`、`category`、`eventName`、`message`。时间段类记录还包含 `operationId`、`phase`、`startedAt`、`finishedAt`、`durationMilliseconds` 和 `succeeded`。

如果运行日志写入失败，应用不会把操作显示为成功。按钮触发的用户命令、传感器刷新、软件自动 tick、轮询恢复、预设变更和设置保存会在显示成功前等待对应成功/终止日志写入完成；写入失败时，状态栏和最近日志会显示“运行日志写入失败”及底层异常信息。每条 `IpmiCommand/CommandCompleted` 会用子进程真实开始/结束时间写入持久队列，再更新界面最近日志，避免 UI 排队延迟造成错误的并发判断。如果底层 IPMI 命令已经执行，应用不会回滚硬件状态，但也不会显示成功。启动阶段发生未处理异常或重复启动时，应用会写入：

```text
%LocalAppData%\DellR730xdFanControlCenter\startup-error.log
```

WebView2 图表运行时用户数据写入：

```text
%LocalAppData%\DellR730xdFanControlCenter\WebView2
```

该目录用于 WebView2 本地运行状态，不包含 iDRAC 密码。创建失败会显示真实图表加载错误，不会改写到 Release 解压目录旁边的 `DellR730xdFanControlCenter.exe.WebView2`。

当前没有自动保留期或自动清理策略。长期轮询会持续增长当天 JSONL 文件；请根据自己的运维习惯手动归档或删除旧日志。提交 issue、截图或日志前，请检查其中是否包含你的 iDRAC 地址、用户名、主机名、路径、预设命名、传感器输出或其他不希望公开的环境信息。不要公开 iDRAC 密码、VPN 地址、内网网段、安全策略、资产编号或其他敏感信息。

## iDRAC 权限

本应用需要能读取 SDR 并发送 Dell OEM raw IPMI 命令的 iDRAC 账号。受限账号可能出现以下情况：

- `mc info` 成功，但 `raw 0x30 ...` 控制命令失败。
- `sdr elist` 成功，但风扇模式切换失败。
- 命令返回权限或认证错误。

建议为此应用使用最小必要权限的 iDRAC 账号，并避免复用高价值管理密码。若你不确定权限边界，请先在隔离网络或测试窗口中验证。

## 网络边界

IPMI over LAN 应只暴露在可信管理网络中。不要把 iDRAC/IPMI 直接暴露到公网。建议：

- 使用独立管理 VLAN 或受控内网。
- 限制能访问 iDRAC 的来源 IP。
- 使用强密码并定期轮换。
- 关闭不需要的管理协议。
- 在不可信网络中不要保存密码。

## 操作安全

风扇控制会改变服务器散热能力。手动低转速可能导致 CPU、硬盘、PCIe 卡、电源、内存或机箱内部温度升高。每次调速后请持续观察：

- CPU 温度和 CPU 使用率。
- Inlet / Exhaust 温度。
- Fan 1-6 RPM。
- 硬盘、背板、线缆在位和冗余状态。
- 功耗、电压、电流。
- iDRAC 告警和系统事件。

当服务器负载未知、环境温度较高、硬盘数量较多、传感器异常或你无法持续观察时，应使用 Dell 自动模式。总览快速操作和托盘菜单中的“还原戴尔出厂设置转速”会在等待 IPMI 锁前停止正在运行的软件自动策略，然后发送 Dell 自动模式命令，把风扇控制权交还给 iDRAC/BMC 固件；它不是手动 10%。

手动百分比控制分两条命令执行：先进入手动模式，再设置目标百分比。仅当第一条已确认成功、第二条失败时，应用会立即发送一次 `raw 0x30 0x30 0x01 0x01`，尝试把控制权交回 Dell 固件。它不会重试失败的百分比命令。恢复成功后，用户请求仍显示为失败，界面和 `LastRunningPresetId` 对齐到 `dell-auto`；恢复也失败时，日志和错误栏会保留百分比失败与恢复失败，且必须把 BMC 可能仍处于手动模式视为需要人工检查的硬件风险。若进入手动模式本身失败，因为没有确认硬件已切换，应用不会额外发送恢复命令。

横幅实时摘要和总览顶部硬件摘要只用于显示最近一次成功 SDR 刷新的概览值。横幅“实时温度”是当前温度传感器读数的平均摘要，不是历史最高值，也不参与软件恒温或紧急自动保护判断；实际控制仍使用软件恒温策略中说明的 CPU/最高温度口径。每个横幅大值下方会按实际返回数量显示该类别的全部具体传感器小项，但仍不能替代完整的“传感器”表。首次刷新前或缺少某类读数时，横幅和总览电气摘要会显示“等待刷新”或无读数。横幅与硬件卡片颜色是界面提示：正常绿色、接近风险黄色、偏离明显橙色、危险红色；这些阈值不会改变风扇控制命令，也不替代 iDRAC 告警、完整传感器表或紧急自动保护。风扇卡片的旋转动画只反映当前 RPM 读数的视觉节奏，不是控制回路的一部分。做风扇调速或排障时仍应查看温度看板、风扇 RPM 看板、功耗与状态看板以及 iDRAC 告警。

## 传感器图标与动画边界

1.0.14 在总览的温度、风扇 RPM、功耗与状态看板中使用语义图标和 Windows Composition 动画。图标覆盖温度、CPU / MEM / IO / SYS 使用率、功耗、电压、电流、入侵、风扇/电源冗余、CMOS / ROMB / BBU 电池、硬盘、RAID/PERC 控制器、缓存、USB 过流和电源策略。normal、information、inactive、unavailable、warning、critical 和 stale 还有非颜色 badge；读屏名称包含数值、状态和断开信息。

- 风扇卡保留原四叶十字形并围绕中心旋转；导航栏使用静态圆形外框和四片弯曲叶片，不再呈对角 X 形，也不表示实时 RPM。0 RPM 停止；只有最近成功 SDR 中的正 RPM 会启动看板旋转。RPM 线性映射到每秒转数，低转速端单圈周期上限为 `5.2 s`，18000 RPM 及以上为 `0.11 s`，3600 RPM 稍快于 3480 RPM。变速只更新播放速率，保留当前旋转相位。
- 温度和 CPU / MEM / IO / SYS level、已知 `Voltage N` 表盘只在真实新样本变化时做一次过渡，不在轮询间隔中生成中间值。已知电压表盘用 `190..260 V`；只凭 `Volts` / `Amps` / `Watts` 单位识别的厂商传感器保留图标，但使用 Information 和静态中性样式；适用电压表盘时，指针位于中位。这类传感器不套用 230 V 或电流/功耗阈值。未识别但状态为 `ok` 的真实类型使用通用正常图标，明确异常状态仍优先。
- 展示层只接受真实有效 SDR 行：`status=ok` 的数值和离散状态会保留，真实 `0%` 不会被误删；`ns`、`na`、`No Reading`、`Disabled`、`Not Available`、`N/A`、`Unknown` 和非有限数值不会进入看板、传感器明细或图表。warning/critical/failure/fault/degraded/lost 等真实告警即使没有数值也保留。BBU 容量、缓存命中率、RAID 性能或其他指标只有当前 `sdr elist` 返回有效行时才显示；没有有效行时不创建默认值、假卡片或替代读数。
- 只有正电流显示流动，只有正功耗显示活动。正常 health 状态静止，warning / critical 才显示状态脉冲。停止轮询、断开连接、stale、0 RPM、disabled 或无读数都会停止动画，并保留可识别的时钟/状态形状。Windows 关闭动画、高对比度或窗口隐藏到托盘时也静止；恢复可见且允许动画后继续。高对比度使用系统前景色和完整不透明度。
- 解析器先匹配精确原始 SDR key，再按单位和真实名称模式回退；异常状态码优先，未知范围不猜测，未登记但有效的名称保留 BMC 原文。Dispatcher 回调或 Composition 更新失败会通过 `VisualUpdateFailed` 进入主页面 `ShowFailure`，打开错误栏并写运行日志，不会把失败标记为动画成功；没有错误处理器时会显式抛出异常，未处理时写入 `%LocalAppData%\DellR730xdFanControlCenter\startup-error.log`，进程可能终止。筛选发生在 `Sensors` 的展示副本上，不改变自动策略使用的原始 SDR 集合，也不改变 IPMI 命令、Dell raw 风扇控制、轮询、自动策略或设置。

该显示层由 `Models/SensorReadingAvailability.cs`、`Models/DashboardSensorPresentation.cs`、`Models/DashboardSnapshotFreshness.cs`、`Controls/DashboardSensorIcon.xaml(.cs)` 和 `MainPage.xaml` 实现。可执行 `dotnet run --project .\Tests\PresetModelTests\PresetModelTests.csproj -c Release` 和 `dotnet build .\DellR730xdFanControlCenter.csproj -c Release -p:Platform=x64` 验证有效性、分类/边界规则与 x64 Release 构建。这些动画不是风扇控制回路或安全告警；iDRAC 告警和界面中的有效传感器明细仍需结合判断。BMC/固件名称差异可能让传感器落入 unit-only 或 generic 路径，更新速度也受完整 `sdr elist` 耗时限制。当前本机验证范围不表示所有 iDRAC 固件、Windows 缩放比或高 DPI 组合都经过人工验证。

## 单风扇目标编号风险

单风扇控制默认关闭。单风扇命令里的 `0x00-0x05` 是固件 raw 命令中的目标编号，用来选择风扇，不是风扇转速；`0x00` 不是 `0%` 转速。本机 R730xd/iDRAC 2.82 实测 Fan 1 目标编号 `0x00` 没有单独控制 Fan 1，而是让全部风扇高转。不同固件可能有不同映射。1.0.15 将页面上的整行黄色说明收为紧凑锁定/高风险状态，但风险图标 ToolTip 和读屏 HelpText 仍保留完整目标编号说明；这只是减少页面占用，不代表风险降低。启用单风扇控制前必须确认：

- 你的 iDRAC 固件接受该 raw 命令。
- 目标编号与实际风扇一一对应。
- 操作后 RPM 和温度符合预期。
- 能随时切回 Dell 自动模式。

如果行为不符合预期，请立即恢复 Dell 自动模式。

## 软件恒温策略风险

软件恒温策略依赖 BMC SDR 传感器读取和 CPU 温度识别。当前逻辑优先使用名称包含 `CPU` 的温度传感器；如果没有 CPU 命名行，则使用所有温度传感器中的最高值。找不到温度传感器会报错。

该策略不是固件级实时控制回路。它受以下因素影响：

- iDRAC 返回 SDR 的速度。
- 网络延迟。
- 命令超时设置。
- 轮询间隔设置。
- 传感器命名和固件输出格式。

默认轮询间隔为 1 秒，保存时允许 1 秒及以上的值。该值只是触发轮询 tick 的频率，不保证 iDRAC 能以相同频率返回完整 SDR。应用同一时刻只允许一个 IPMI 操作；上一轮 SDR 读取、软件自动 tick 或其他 IPMI 命令仍在执行时，新的后台 tick 会跳过，不会启动新的 `ipmitool` 进程或建立新的 RMCP+ 会话。启动软件自动或切换曲线预设时，首轮 tick 必须成功后才启动后台计时器；首轮失败、IPMI 锁正忙或运行日志终止记录写入失败都会停止启动并显示根因，不会排队增加处理时间。软件自动或曲线自动运行时，普通传感器轮询不会再独立启动第二条 `sdr elist` 链路；自动策略 tick 的 SDR 结果负责刷新界面和历史点。若自动策略成功读取 SDR 但计算出的目标百分比不变，本轮会刷新传感器和历史点、关闭旧的顶部失败提示，并跳过重复 raw 风扇命令；跳过记录写入失败仍按失败处理。自动计时器已经运行后，如果在计算出风扇目标前发生 SDR 读取失败，应用会显示并记录真实失败，但不会清空当前自动策略或已保存运行预设；下一次计划自动 tick 会重新执行真实 SDR 读取。后台传感器轮询的实际命令若返回 `Unable to establish IPMI v2 / RMCP+ session`、超时或非零退出，应用会停止当前轮询并显示失败原因，然后按当前轮询间隔持续执行真实重连（`mc info` + `sdr elist`）；重连成功才恢复轮询，重连失败保持断开并等待下一次重连，不会静默降级、把失败显示为成功或写入无真实 SDR 来源的历史点。Dell 风扇控制 raw 命令不会自动重连或重试，非零即失败。

达到紧急温度阈值时，应用会发送 Dell 自动模式命令，让 BMC 接管风扇。该动作本身仍依赖 IPMI 命令能成功执行。命令成功后，应用会停止软件自动计时器、清空活动曲线/软件自动状态、把 Dell 自动持久化为当前运行模式，并显示明确警告。

## 曲线预设风险

曲线预设是软件自动策略的一种规则，不会写入 iDRAC/BMC 固件。温度曲线在每次软件自动 tick 中读取 SDR、解析 CPU 温度、按用户保存的温度-风扇点插值，并发送全部风扇百分比命令。功耗曲线同样读取 SDR，但用单位包含 `Watts` 或名称包含 `Pwr Consumption` 的功耗传感器作为曲线输入；它仍会先解析 CPU 温度并检查紧急自动阈值。曲线点通过风扇控制页的图表点击和点位数字控件维护；1.0.15 的悬停读数固定在图表外，十字线不可命中，标题说明和 ASCII 预览被移除，但无效点位仍在固定读数栏显示真实原因。“平滑曲线”只是软件端插值，不会让 iDRAC/BMC 记住任何固件级风扇曲线。

曲线预设的安全边界：

- 曲线点只控制全部风扇百分比，不控制单个风扇。
- 保存或切换时会校验至少 2 个点。温度曲线校验温度 `-40` 到 `125` °C、百分比 `0` 到 `100`、温度不重复；功耗曲线校验功耗 `0` 到 `1200` W、百分比 `0` 到 `100`、功耗点不重复。
- 点位列表和 `SmoothCurve` 平滑开关保存在本地 `settings.json`；删除、编辑或保存预设会影响后续软件 tick 的百分比计算，但不会修改服务器固件配置。
- 输入值低于第一个点时使用第一个点，高于最后一个点时使用最后一个点；如果最后一个点设置得过低，持续高负载时仍可能散热不足。
- 功耗曲线不能替代温度保护。功耗低不代表局部硬件温度安全；硬盘、PCIe 卡、内存或进/排风温度仍可能升高。若本轮 SDR 缺少功耗读数，功耗曲线会显示失败并停止本轮下发，不会用默认值或上次功耗值继续控制。
- 开启平滑后，温度和功耗曲线都使用单调三次 Hermite 插值与 Fritsch-Carlson 切线约束。结果精确经过控制点，不超过相邻端点百分比，平坦区间保持平坦，两个点时退化为线性插值；图表预览和自动策略使用同一求值器。最终百分比仍按现有规则四舍五入并钳制到 `0-100%`。该算法不改变紧急阈值，也不会给过低的点位增加安全余量。
- 达到紧急温度阈值时，应用会优先发送 Dell 自动模式命令。该保护仍依赖 SDR 读取、CPU 温度识别、网络和 IPMI 命令执行成功。Dell 自动命令成功后，应用会停止软件自动计时器、清空自动模式状态，并把 Dell 自动持久化为当前运行模式。
- 手动全部风扇、单风扇、内置恢复手动预设、Dell 自动、总览/托盘还原戴尔出厂设置转速和停止自动都会清除当前曲线状态。用户主动的手动和 Dell 自动命令会在等待 IPMI 锁前停止软件自动策略，避免旧的队列命令或后台 tick 覆盖新的用户命令。删除正在运行的曲线预设、停止自动、已经计算出目标后的 raw 风扇命令失败和紧急 Dell 自动保护也会停止软件自动计时器并清空自动模式状态；已经运行的自动 tick 在计算目标前读取 SDR 失败不会清空该自动状态。

建议先在低负载和有人值守的窗口验证曲线。不要把曲线的最后一个温度点设得低于实际高负载温度，也不要把最后一个风扇百分比设得过低。若图表、传感器状态或 iDRAC 告警显示异常，请立即恢复 Dell 自动模式。

## 失败处理原则

本项目要求失败显式暴露：

- 缺少内置 `ipmitool.exe` 时直接报错。
- iDRAC 主机、用户名、密码为空时直接报错。
- 命令超时时终止进程并显示超时原因；如果该超时发生在后台传感器轮询，应用随后持续执行真实重连，直到轮询恢复或用户取消轮询。
- `ipmitool` 返回非零退出码时立即显示本次 stdout/stderr；Dell 风扇控制 raw 命令不做固定重试。
- SDR 输出为空时直接报错。
- 缺少 UI 翻译时直接报错。

除上述手动模式残留风险的一次性 Dell 自动安全恢复外，除非未来有明确设计并在文档中说明触发条件、用户可见行为和风险，否则不要加入会掩盖真实失败的静默 fallback、默认成功、吞异常或无真实来源数据。

## 报告安全问题

报告问题时请包含：

- 应用版本或提交号。
- Windows 版本。
- iDRAC 固件版本。
- 触发步骤。
- 可公开的错误消息。
- 相关日志中已经脱敏的片段。

请不要包含：

- iDRAC 密码。
- 未脱敏的内网地址规划。
- VPN、堡垒机或防火墙策略细节。
- 私有资产编号或主机名。
