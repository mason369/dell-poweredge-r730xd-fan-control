# 曲线编辑器与图表交互设计

## 决策

用户在 2026-07-10 明确选择推荐方案，不使用浏览器可视化稿。实现范围包括温度曲线、功耗曲线、曲线悬停交互、页面说明文案和单风扇安全状态。发布版本由 1.0.14 提升为 1.0.15。

## 目标

- 曲线图直接承担编辑反馈，不再用大段文字解释基本操作。
- 鼠标悬停信息不覆盖指针附近的点位，也不拦截点击或拖动。
- 平滑模式在相邻控制点之间保持连续、单调且无过冲；线性模式保持原行为。
- 温度曲线和功耗曲线使用同一套视觉规则和插值约束。
- 页面只保留状态、验证失败和必要安全边界，移除重复的功能说明。
- 单风扇控制仍明确显示固件风险，但不再占用整行黄色 InfoBar。

## 曲线编辑器

每个编辑器保留名称、平滑开关、重置、图表、点位数值编辑和保存入口。图表高度提高，采用稳定的绘图区边距、主次网格、轴刻度、面积底色、清晰曲线和控制点。当前悬停或拖动的控制点使用外环强调，不能改变布局尺寸。

图表下方增加固定高度的读数栏。悬停时显示当前横轴值、计算出的风扇百分比和最近控制点；未悬停时显示曲线点数、输入范围和输出范围。读数栏不覆盖画布，画布上的十字线、标记和高亮全部设置为不可命中。

删除右侧 ASCII 预览和标题下的操作说明。数值列表仍用于精确输入；错误只在存在无效点、重复横轴值、越界值或点数不足时显示。

## 平滑算法

平滑模式使用单调三次 Hermite 插值。切线根据相邻区间斜率计算，并使用单调性约束限制，避免在控制点之间超过相邻端点范围。首尾控制点必须精确命中；相等输出形成水平区间；只有两个控制点时采用线性插值。最终风扇百分比仍按现有最小值、最大值和整数规则钳制。

温度和功耗路径共用同一插值函数。图表预览和自动策略计算必须调用同一逻辑，避免“图上是一条曲线，实际下发是另一条曲线”。关闭平滑时继续使用现有分段线性算法。

## 文案收敛

移除总览图表、温度看板、温度曲线、功耗曲线和软件恒温策略标题下的功能说明。字段名、按钮、模式状态、数据来源、连接状态和错误信息保留。

单风扇区域使用紧凑状态行：未启用时显示锁定状态；启用时显示高风险状态。详细的 0x00-0x05 目标选择字节说明放入风险图标的 ToolTip 和辅助功能 HelpText，并保留设置入口。风险内容不能删除，也不能把启用状态显示为成功或正常。

## 失败行为与安全边界

- 无效曲线不保存、不启动自动策略，并显示真实验证原因。
- 插值函数遇到非有限值、无序点或重复横轴值时不得生成看似有效的结果；上层继续使用现有验证流程阻止执行。
- 本次不修改 IPMI 排队、轮询、Dell raw 命令、首轮真实自动 tick 或紧急保护逻辑。
- 发布后启动新版前必须确认当前 `ipmitool` 已退出。切换进程会停止旧轮询；不得在活动 IPMI 会话中强制结束。
- GUI 验证优先使用只读传感器轮询。除非恢复既有运行状态确实要求，否则不额外触发风扇 raw 命令。

## 验证

- 模型测试覆盖平滑端点、单调性、无过冲、平坦区间、双点线性退化，以及温度/功耗一致性。
- 源码结构测试覆盖悬停层不可命中、固定读数栏、说明文案和 ASCII 预览移除、单风扇紧凑状态。
- 运行现有全部测试、Release x64 构建、发布脚本和 `git diff --check`。
- 核对 1.0.15 发布目录、zip、exe/dll 版本、应用包清单、WebView2 用户数据位置、运行日志、`startup-error.log`、WebView2 dump、Windows Application Event Log 和发布目录旁的旧 WebView2 文件夹。
- 实际窗口检查曲线页宽窄布局、悬停/拖动无阻挡、页面滚动一致、单风扇状态不产生额外滚动条。
